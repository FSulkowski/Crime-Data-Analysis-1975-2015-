ETL Process

Our group followed the ETL (Extract, Transform, Load) process to manage the crime data. We began by extracting raw crime data from a CSV file containing crime statistics across cities. Next, we used Python and the Pandas library for data transformation, cleaning, and manipulation. This included removing duplicates, handling missing values, and ensuring data consistency. Finally, the transformed data was loaded into a PostgreSQL database, which allowed for structured storage and efficient querying, enabling deeper analysis of crime trends and patterns.

Data Sources

The crime data for this project was initially sourced from Kaggle.com, where we obtained a CSV file containing detailed crime statistics. Ultimately, the data originates from the FBI's Uniform Crime Reporting (UCR) program, specifically the "Offenses Known and Clearances by Arrest" database. Maintained by the National Archives of Criminal Justice Data, this dataset offers a comprehensive overview of crime trends across various cities, providing valuable insights for analysis.

Tools/Technology Used

  • Python: Used for data manipulation and cleaning, leveraging libraries like Pandas for data transformation and analysis.
  • Pandas: A powerful Python library used for data manipulation and analysis, including cleaning, transforming, and analyzing large datasets.
  • PostgreSQL: A relational database management system used to store and query the cleaned and transformed data for efficient analysis.
  • Matplotlib & Seaborn: Python libraries used for data visualization, helping to create clear, informative charts and graphs to represent crime trends.
  • Jupyter Notebooks: An open-source web application used for documenting the analysis process, including writing and executing Python code interactively.
  • Folium: A Python library used for creating interactive maps with leaflet.js, visualizing crime trends across different cities.
  • JavaScript: Used for interactive features on the website, enhancing user experience and dynamic data visualization.
  • Flask: A web framework for Python used to create the web application, enabling seamless integration with the front-end and back-end components.
  • HTML & CSS: Technologies used to build the structure and design of the website, ensuring a user-friendly and visually appealing interface.
  • GitHub: A version control platform used for managing project files, collaborating with the team, and hosting the project repository.
  • Leaflet: A JavaScript library used in combination with Folium for rendering interactive maps, enhancing the presentation of crime data on geographic layouts.
  • Geoapify: A location-based service used to integrate geospatial data into the project, providing features such as geocoding and mapping services for visualizing crime trends.
