ETL Process

Our group followed the ETL (Extract, Transform, Load) process to manage the crime data. We began by extracting raw crime data from a CSV file containing crime statistics across cities. Next, we used Python and the Pandas library for data transformation, cleaning, and manipulation. This included removing duplicates, handling missing values, and ensuring data consistency. Finally, the transformed data was loaded into a PostgreSQL database, which allowed for structured storage and efficient querying, enabling deeper analysis of crime trends and patterns.

Data Sources

The crime data for this project was initially sourced from Kaggle.com, where we obtained a CSV file containing detailed crime statistics. Ultimately, the data originates from the FBI's Uniform Crime Reporting (UCR) program, specifically the "Offenses Known and Clearances by Arrest" database. Maintained by the National Archives of Criminal Justice Data, this dataset offers a comprehensive overview of crime trends across various cities, providing valuable insights for analysis.
